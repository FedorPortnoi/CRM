Российские доступы для CRM

Инструкция для человека без технической подготовки: что открыть, что нажать, какие значения сохранить и что передать Федору.

Для кого этот документ

Документ написан для взрослого делового пользователя, например для маркетинг-директора. Здесь нет программирования и командной строки. Ваша задача - зарегистрировать нужные российские сервисы, сохранить выданные ими значения и передать их Федору безопасным способом.

Что нельзя делать

Не отправляйте секретные ключи обычным сообщением в Telegram, WhatsApp, email или чат. Лучше передать их через менеджер паролей, во время созвона, или внести в файл на компьютере вместе с Федором.

Проект: CRM
Папка проекта: C:\Users\fedor\crm
Нужный файл для Федора: C:\Users\fedor\crm\.env
Дата подготовки: 17 мая 2026

Что нужно получить

В этот документ включены только российские или Россия-ориентированные сервисы, которые нужны текущей версии CRM. Google, Twilio, Sentry, обычная почта, WhatsApp и Telegram сюда не включены.

Сервис Зачем нужен Что должно быть у Федора в итоге
Яндекс ID / OAuth Подключение Яндекс Календаря к CRM. YANDEX_CLIENT_ID, YANDEX_CLIENT_SECRET, YANDEX_REDIRECT_URI
SMS.ru Отправка SMS клиентам и прием ответов. SMSRU_API_ID, SMSRU_SENDER
Yandex Cloud: Vision и SpeechKit Распознавание визиток и голосовых заметок. YANDEX_API_KEY, YANDEX_FOLDER_ID
Yandex Cloud: PostgreSQL Российская база данных для контактов, сделок, задач и встреч. DATABASE_URL, DIRECT_URL
Yandex Object Storage Хранилище файлов: вложения, аудио, выгрузки. S3_BUCKET, S3_ENDPOINT, S3_REGION, AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY

Как работать с инструкцией

Делайте строго по одному разделу. Не переходите к следующему, пока текущий сервис не создан, значения не сохранены и не отмечены в чек-листе.

Как сохранять значения

Каждый сервис выдаст несколько строк. Их не нужно понимать. Их нужно аккуратно скопировать, подписать понятным названием и сохранить.

Удобный формат заметки

CRM - доступы

1. Яндекс Календарь
YANDEX_CLIENT_ID = ...
YANDEX_CLIENT_SECRET = ...
YANDEX_REDIRECT_URI = ...

2. SMS.ru
SMSRU_API_ID = ...
SMSRU_SENDER = ...

3. Yandex Cloud
YANDEX_API_KEY = ...
YANDEX_FOLDER_ID = ...

4. База данных Yandex PostgreSQL
DATABASE_URL = ...
DIRECT_URL = ...

5. Yandex Object Storage
S3_BUCKET = ...
S3_ENDPOINT = ...
S3_REGION = ...
AWS_ACCESS_KEY_ID = ...
AWS_SECRET_ACCESS_KEY = ...

Если сервис показывает ключ только один раз, не закрывайте страницу, пока ключ не сохранен. Если закрыли и ключ пропал, обычно проще удалить старый ключ и создать новый.

1Яндекс Календарь

Это нужно, чтобы встречи из CRM попадали в Яндекс Календарь пользователя. Здесь создается маленькое приложение в Яндексе. Через него пользователь дает CRM разрешение работать с календарем.

Куда зайти

Откройте: https://oauth.yandex.ru/

Что нажимать

  1. Войдите в Яндекс аккаунт, который будет владельцем CRM-доступов.
  2. Нажмите создание нового приложения.
  3. Если появится вопрос Какое приложение хотите создать?, выберите Для авторизации пользователей.
  4. Название приложения: CRM Calendar или CRM Календарь.
  5. Описание: Подключение Яндекс Календаря к CRM.
  6. Тип приложения: выберите вариант для сайта, веб-сервиса или веб-приложения.
  7. В поле адреса возврата вставьте:
    http://localhost:3000/api/v1/calendar/sync/yandex/callback
  8. В правах доступа найдите календарь. Ищите слова Календарь, Calendar или CalDAV.
  9. Выберите право, которое разрешает читать и изменять календарь.
  10. Если отдельно попросят доступ к логину пользователя, разрешите минимальный доступ. CRM использует логин, чтобы понять адрес календаря.
  11. Нажмите создать приложение.

Что скопировать

Как называется на сайте Как подписать для Федора
Client ID / ID приложения YANDEX_CLIENT_ID
Client secret / Пароль приложения / Секрет приложения YANDEX_CLIENT_SECRET
Redirect URI / Callback URL / Адрес возврата YANDEX_REDIRECT_URI

Значение адреса возврата

YANDEX_REDIRECT_URI=http://localhost:3000/api/v1/calendar/sync/yandex/callback

Чек-лист

  • Приложение в Яндексе создано.
  • Выбран вариант для авторизации пользователей.
  • Адрес возврата вставлен точно, без лишних символов.
  • Права календаря выбраны.
  • YANDEX_CLIENT_ID сохранен.
  • YANDEX_CLIENT_SECRET сохранен.

2SMS.ru

Это российский сервис для SMS. Он нужен, чтобы CRM отправляла SMS клиентам и принимала ответы. В старых материалах может встречаться Twilio. Для России сейчас нужен SMS.ru, не Twilio.

Куда зайти

Откройте: https://sms.ru/

Что нажимать

  1. Зарегистрируйтесь или войдите в SMS.ru.
  2. Подтвердите телефон и email, если сервис попросит.
  3. Откройте личный кабинет.
  4. Найдите раздел API, Настройки API, Программистам или Интеграция.
  5. Найдите значение api_id. Это главный код для CRM.
  6. Скопируйте api_id.
  7. Пополните баланс на небольшую сумму для тестовых SMS.
  8. Если нужна отправка от имени бренда, найдите раздел с именем отправителя и подайте имя на согласование.

Что скопировать

Как называется на сайте Как подписать для Федора
api_id SMSRU_API_ID
Имя отправителя, если согласовано SMSRU_SENDER

Если имя отправителя еще не готово

Это не блокирует начало работы. Для тестов можно оставить: 

SMSRU_SENDER=CRM

Про входящие SMS

Когда CRM будет размещена на настоящем сервере, в SMS.ru нужно будет указать адрес, куда отправлять входящие SMS. Сейчас, на этапе получения доступов, достаточно сохранить SMSRU_API_ID.

Чек-лист

  • Аккаунт SMS.ru создан.
  • Телефон/email подтверждены.
  • SMSRU_API_ID сохранен.
  • Баланс пополнен для теста.
  • Если нужно брендированное имя отправителя, заявка отправлена.

3Yandex Cloud: визитки и голос

Этот пункт нужен для двух функций CRM: загрузить фото визитки и получить текст, а также записать голосовую заметку и получить текст. Для обеих функций нужен один и тот же ключ Яндекс Облака.

Куда зайти

Откройте: https://console.yandex.cloud/

Сначала подготовьте Яндекс Облако

  1. Войдите в Яндекс Облако.
  2. Если облака еще нет, создайте облако.
  3. Если Яндекс просит подключить оплату, подключите платежный аккаунт. Без этого многие сервисы не заработают.
  4. Создайте или выберите папку для CRM. Удобное название: crm или crm-prod.
  5. Скопируйте ID папки. Он виден на странице папки под названием или в информации о папке.

Потом создайте отдельного пользователя для CRM внутри Яндекс Облака

В Яндексе это называется сервисный аккаунт. Можно думать о нем как о служебном пользователе, который работает не руками человека, а от имени CRM.

  1. В Яндекс Облаке откройте Identity and Access Management.
  2. Откройте Сервисные аккаунты.
  3. Нажмите Создать сервисный аккаунт.
  4. Название: crm-api.
  5. Создайте аккаунт.
  6. Назначьте ему права для распознавания изображений и речи. Ищите роли, связанные с Vision и SpeechKit.
  7. Если интерфейс непонятен, остановитесь и позовите Федора. Роли лучше выбрать аккуратно.

Создайте API-ключ

  1. Откройте сервисный аккаунт crm-api.
  2. Найдите кнопку Создать новый ключ.
  3. Выберите API-ключ.
  4. Описание: CRM Vision Speech.
  5. Если есть выбор ограничений, добавьте доступ к распознаванию изображений и речи.
  6. Создайте ключ.
  7. Сразу скопируйте ключ. Яндекс показывает его полностью только при создании.

Что скопировать

Что вы видите Как подписать для Федора
API-ключ YANDEX_API_KEY
ID папки YANDEX_FOLDER_ID

Чек-лист

  • В Яндекс Облаке есть платежный аккаунт.
  • Создана папка CRM.
  • YANDEX_FOLDER_ID сохранен.
  • Создан сервисный аккаунт crm-api.
  • Создан API-ключ.
  • YANDEX_API_KEY сохранен.

4Yandex Cloud: база данных

База данных - это место, где CRM хранит всю рабочую информацию: контакты, сделки, задачи, встречи и сообщения. Для русской версии база должна быть в российской инфраструктуре.

Куда зайти

Откройте: https://console.yandex.cloud/

Что нажимать

  1. Откройте папку CRM в Яндекс Облаке.
  2. Найдите сервис Managed Service for PostgreSQL.
  3. Нажмите создание нового кластера.
  4. Название: crm-postgres.
  5. Выберите российский регион, который предлагает Яндекс.
  6. Для начала выберите небольшой тариф. На старте не нужен большой сервер.
  7. Создайте пользователя базы. Имя: crm_user.
  8. Создайте сложный пароль. Сохраните его отдельно.
  9. Создайте базу данных. Имя: crm.
  10. На странице подключения скопируйте адрес сервера и порт.

Что нужно передать Федору

Федору нужны не отдельные куски, а две готовые строки подключения. Если вам неудобно собирать их самой, передайте Федору четыре значения: адрес сервера, порт, имя базы, имя пользователя и пароль.

Что найти в Яндексе Пример
Адрес сервера rc1a-...mdb.yandexcloud.net
Порт 6432 или тот, который показывает Яндекс
Имя базы crm
Пользователь crm_user
Пароль тот, который вы создали

Как будут выглядеть строки для Федора

DATABASE_URL="postgresql://crm_user:ПАРОЛЬ@АДРЕС:ПОРТ/crm?sslmode=require"
DIRECT_URL="postgresql://crm_user:ПАРОЛЬ@АДРЕС:ПОРТ/crm?sslmode=require"

Чек-лист

  • Кластер PostgreSQL создан.
  • База crm создана.
  • Пользователь crm_user создан.
  • Пароль сохранен.
  • Адрес сервера и порт сохранены.
  • Старые значения базы не удалялись без копии.

Это самый чувствительный пункт. Если Яндекс спрашивает про сеть, SSL, доступ снаружи или белые IP-адреса, лучше остановиться и позвать Федора.

5Yandex Object Storage

Это хранилище файлов. В CRM туда можно складывать вложения, выгрузки, аудио и другие файлы. В настройках проекта названия полей похожи на Amazon, но использовать будем Яндекс.

Куда зайти

Откройте: https://console.yandex.cloud/

Создайте хранилище

  1. Откройте папку CRM в Яндекс Облаке.
  2. Найдите сервис Object Storage.
  3. Нажмите Создать бакет.
  4. Имя: crm-uploads. Если имя занято, добавьте короткое окончание, например crm-uploads-fedor.
  5. Не включайте публичный доступ, если Яндекс это предлагает. Файлы CRM не должны быть открыты всем.
  6. Создайте бакет.

Создайте ключи для доступа к файлам

  1. Откройте Identity and Access Management.
  2. Создайте или откройте сервисный аккаунт, например crm-storage.
  3. Дайте ему права на Object Storage для папки или нужного бакета.
  4. Откройте сервисный аккаунт.
  5. Нажмите Создать новый ключ.
  6. Выберите Статический ключ доступа.
  7. Сразу скопируйте обе части ключа.

Что скопировать

Что вы видите Как подписать для Федора
Имя бакета S3_BUCKET
Endpoint Яндекса S3_ENDPOINT=https://storage.yandexcloud.net
Регион S3_REGION=ru-central1
Идентификатор статического ключа AWS_ACCESS_KEY_ID
Секрет статического ключа AWS_SECRET_ACCESS_KEY

Чек-лист

  • Бакет создан.
  • Публичный доступ не включен.
  • S3_BUCKET сохранен.
  • AWS_ACCESS_KEY_ID сохранен.
  • AWS_SECRET_ACCESS_KEY сохранен.
  • S3_ENDPOINT=https://storage.yandexcloud.net записан.
  • S3_REGION=ru-central1 записан.

Что не нужно получать сейчас

Сервис Почему не нужен сейчас
Google Calendar Заменен на Яндекс Календарь.
Twilio Заменен на SMS.ru для России.
Sentry Не российский обязательный доступ. Можно настроить позже.
Обычная email-рассылка / SMTP Не входит в список российских обязательных доступов для первого запуска.
WhatsApp Business Отложено на следующую версию.
Telegram Bot Отложено на следующую версию.
Оплата Будет отдельным шагом, когда будет готова юридическая часть.

Итоговый лист для Федора

Когда все готово, у вас должен быть заполнен такой список. Пустые места заполняются реальными значениями. 

YANDEX_CLIENT_ID=
YANDEX_CLIENT_SECRET=
YANDEX_REDIRECT_URI=http://localhost:3000/api/v1/calendar/sync/yandex/callback

SMSRU_API_ID=
SMSRU_SENDER=

YANDEX_API_KEY=
YANDEX_FOLDER_ID=

DATABASE_URL=
DIRECT_URL=

S3_BUCKET=
S3_ENDPOINT=https://storage.yandexcloud.net
S3_REGION=ru-central1
AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=

Когда писать Федору

После каждого раздела можно написать: "Готово: Яндекс Календарь", "Готово: SMS.ru" и так далее. Сами секретные значения лучше не отправлять обычным сообщением.
