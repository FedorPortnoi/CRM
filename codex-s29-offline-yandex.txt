Yandex OAuth endpoints:
  Authorization URL: https://oauth.yandex.ru/authorize
  Token URL: https://oauth.yandex.ru/token
  User info URL: https://login.yandex.ru/info?format=json
    → returns { login: string, ... } — use login as the Yandex username

Yandex CalDAV:
  Base URL: https://caldav.yandex.ru
  Calendar URL: https://caldav.yandex.ru/calendars/{username}/{calendar_slug}/
  Event URL:    https://caldav.yandex.ru/calendars/{username}/{calendar_slug}/{event_uid}.ics
  Auth header:  Authorization: OAuth {access_token}   (NOT Bearer — Yandex uses "OAuth" prefix)

For syncYandexEventForUser(userId, eventId):
  1. Get sync record from DB (provider = 'yandex')
  2. Fetch calendar event from DB
  3. Build iCal content as a string (VCALENDAR/VEVENT format — see below)
  4. PUT to CalDAV event URL; create uid from event.id (use the CRM UUID as the CalDAV uid)
  5. If PUT succeeds and ext_event_uid was not set, update DB: { ext_event_uid: event.id, ext_calendar_uid: username/slug }

For deleteYandexEventForUser(userId, eventId):
  1. Get event from DB, check ext_event_uid exists
  2. Get sync record
  3. DELETE from CalDAV URL

iCal string builder (minimal VEVENT — no dependency needed):
  function buildIcal(event: CalendarEvent): string {
    const fmt = (d: Date) => d.toISOString().replace(/[-:]/g, '').replace('.000', '');
    return [
      'BEGIN:VCALENDAR',
      'VERSION:2.0',
      'PRODID:-//CRM//CRM//EN',
      'BEGIN:VEVENT',
      `UID:${event.id}`,
      `DTSTART:${fmt(event.start_time)}`,
      `DTEND:${fmt(event.end_time)}`,
      `SUMMARY:${event.title}`,
      event.location ? `LOCATION:${event.location}` : '',
      event.description ? `DESCRIPTION:${event.description.replace(/\n/g, '\\n')}` : '',
      `DTSTAMP:${fmt(new Date())}`,
      'END:VEVENT',
      'END:VCALENDAR',
    ].filter(Boolean).join('\r\n');
  }
